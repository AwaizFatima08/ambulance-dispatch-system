# Ambulance Dispatch System — Version Roadmap

## Version 1 — Local Prototype Foundation

### Objective
Build a working local ambulance dispatch prototype for reception-side use with expandable architecture.

### Included Scope
- Flutter Android application
- Reception dashboard
- Create dispatch screen
- Dispatch model
- DispatchService memory storage
- Dispatch card display
- Local device testing
- Status lifecycle implementation
- Basic driver terminal
- Maintenance framework

### Excluded from Version 1
- Cloud database
- Multi-device synchronization
- VoIP integration
- GPS tracking
- Analytics engine
- ERP or HRMS integration

# Scope Control Rules

1. Version-1 scope must remain narrow and controlled.
2. New ideas must be documented before coding.
3. Deferred ideas remain in roadmap until scheduled.
4. Each milestone must be documented in milestone log.
5. Major version upgrades require roadmap update.
